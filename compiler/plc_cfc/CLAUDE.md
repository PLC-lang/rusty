# plc_cfc

From-scratch implementation of CFC (Continuous Function Chart) support for this compiler,
loosely based on the IEC 61131-10 XML exchange format. "Loosely" means we follow the
standard's vocabulary but the fixtures might not be 100% correct yet as we're still waiting for another
team to decide for the final format. The XML format is generated by the IDE team, we take that XML output
and transpile it into AST nodes in the compiler. As such we are dependant on the format of the IDE team
which are not yet done as such the format may change in the future.

## Relationship to the legacy `plc_xml` crate

`plc_xml` is the old CFC implementation, built around the PLCopen TC6 XML format
(`http://www.plcopen.org/xml/tc6_0201`). It exists and works to some degree, and it can be
used as a *reference* (e.g. how a CFC frontend integrates with the rest of the compiler) —
but **do not derive too much from it**: its design is considered poor and imitating it
would degrade this implementation. When in doubt, design from the IEC 61131-10 standard and
this crate's fixtures, not from `plc_xml`.

## Layout

- `assets/IEC61131/cfc/` — the IEC 61131-10 standard (PDF), the official XSD
  (`examples/IEC61131_10_Ed1_0.xsd`) and the official example document
  (`examples/IEC61131_10_Ed1_0_Example.xml`).
- `fixtures/` — hand-written `.cfc`/`.st` fixture pairs the implementation is developed
  against (one directory per example).
