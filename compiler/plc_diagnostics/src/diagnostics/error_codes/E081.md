# CFC Connection References No Producer

Every wire in a CFC network connects a consumer — a block input, a data sink, a return condition — back to
the value that some other object produces, matched by connection id. This error reports a wire whose
referenced value is produced by nothing in the network: a dangling connection.

This usually means the `.cfc` file is out of sync with the graphical editor. Common causes are:

- a stale reference (`refConnectionPointOutId`) pointing at an id that no object produces;
- a connector whose continuation feeds a consumer, but the connector itself has no incoming wire, so the
  "virtual wire" starts from nothing;
- a connector/continuation cycle whose labels loop back to no real producer.

Erroneous network (the sink reads a connection that no object produces):

```text
   localA  --(2)                 (nothing produces 999)
                    result  --(999?)-->
```

Erroneous network (the continuation "relay" feeds the sink, but its connector has no source):

```text
   (no source)-->[ Connector "relay" ]

   [ Continuation "relay" ]-->  result
```

To fix, reconnect the wire in the editor so it originates at an object that produces a value, or delete the
dangling consumer.
