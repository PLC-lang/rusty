# Missing Text Declaration in a CFC POU

A CFC (graphical) POU is compiled from two parts: a text declaration that provides the POU header and its
variable blocks, and the graphical network that provides the body. Without the text declaration there is no
interface to compile the network against, so the whole file cannot be processed.

This usually means the `.cfc` file is incomplete or was edited by hand and lost its declaration.

Erroneous file (declaration omitted):

```xml
<ppx:Program name="mainProgram">
    <ppx:MainBody>
        <ppx:BodyContent xsi:type="ppx:FBD">
            <ppx:Network/>
        </ppx:BodyContent>
    </ppx:MainBody>
</ppx:Program>
```

To fix, provide the text declaration, for example:

```xml
<ppx:Program name="mainProgram">
    <ppx:AddData>
        <ppx:Data name="http://www.bachmann.at/xml/PLC" handleUnknown="implementation">
            <textDeclaration>
                <content>PROGRAM mainProgram
VAR
    result : DINT;
END_VAR</content>
            </textDeclaration>
        </ppx:Data>
    </ppx:AddData>
    <ppx:MainBody>
        <!-- ... -->
    </ppx:MainBody>
</ppx:Program>
```
