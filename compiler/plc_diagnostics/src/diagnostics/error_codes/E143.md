# Invalid Evaluation Order in a CFC Network

Every object in a CFC network carries an evaluation priority which decides the order the objects run in.
A block's results only exist once the block has been evaluated, so anything consuming such a result — a
variable, another block's input, or a return condition — must be scheduled *after* the producing block.
This error reports a wire whose consumer would run before (or together with) the block producing its
value.

Erroneous network (the variable at priority `0` reads the result of the block evaluated at priority `1`):

```text
+--- alwaysFive ---+  (1)
|        alwaysFive|--------->  result  (0)
+------------------+

(n) = evaluation priority
```

To fix, adjust the execution order in the editor so the block comes first:

```text
+--- alwaysFive ---+  (0)
|        alwaysFive|--------->  result  (1)
+------------------+
```

A feedback wire — a block consuming its own result, or a cycle of blocks feeding each other — always
triggers this error, because no schedule can evaluate every block of a cycle before its consumers. Break
the cycle by storing the result in a variable and feeding that variable back in a later (or earlier)
part of the network.
