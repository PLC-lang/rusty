# Invalid Expression in a CFC Variable

A variable in a CFC network (a data source or sink) carries its content as free text, and that text is
embedded into the compiled code as-is. Only plain *value* expressions are allowed — a name (`foo`), a
literal (`100`), member or array access (`foo[1].bar`), or compositions of them (`foo + 1`, `-bar`).

Erroneous network:

```text
   conjure() + 5  ------->  result  (0)
```

To fix, draw the call as a block and wire its result instead:

```text
   +--- conjure ---+  (0)
   |        conjure|-------->[ + 5 ]------->  result  (1)
   +---------------+
```
