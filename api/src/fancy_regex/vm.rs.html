21//! Backtracking VM for implementing fancy regexes.
22//!
23//! Read <https://swtch.com/~rsc/regexp/regexp2.html> for a good introduction for how this works.
24//!
25//! The VM executes a sequence of instructions (a program) against an input string. It keeps track
26//! of a program counter (PC) and an index into the string (IX). Execution can have one or more
27//! threads.
28//!
29//! One of the basic instructions is `Lit`, which matches a string against the input. If it matches,
30//! the PC advances to the next instruction and the IX to the position after the matched string.
31//! If not, the current thread is stopped because it failed.
32//!
33//! If execution reaches an `End` instruction, the program is successful because a match was found.
34//! If there are no more threads to execute, the program has failed to match.
35//!
36//! A very simple program for the regex `a`:
37//!
38//! ```text
39//! 0: Lit("a")
40//! 1: End
41//! ```
42//!
43//! The `Split` instruction causes execution to split into two threads. The first thread is executed
44//! with the current string index. If it fails, we reset the string index and resume execution with
45//! the second thread. That is what "backtracking" refers to. In order to do that, we keep a stack
46//! of threads (PC and IX) to try.
47//!
48//! Example program for the regex `ab|ac`:
49//!
50//! ```text
51//! 0: Split(1, 4)
52//! 1: Lit("a")
53//! 2: Lit("b")
54//! 3: Jmp(6)
55//! 4: Lit("a")
56//! 5: Lit("c")
57//! 6: End
58//! ```
59//!
60//! The `Jmp` instruction causes execution to jump to the specified instruction. In the example it
61//! is needed to separate the two threads.
62//!
63//! Let's step through execution with that program for the input `ac`:
64//!
65//! 1. We're at PC 0 and IX 0
66//! 2. `Split(1, 4)` means we save a thread with PC 4 and IX 0 for trying later
67//! 3. Continue at `Lit("a")` which matches, so we advance IX to 1
68//! 4. `Lit("b")` doesn't match at IX 1 (`"b" != "c"`), so the thread fails
69//! 5. We continue with the previously saved thread at PC 4 and IX 0 (backtracking)
70//! 6. Both `Lit("a")` and `Lit("c")` match and we reach `End` -> successful match (index 0 to 2)
71

82/// Enable tracing of VM execution. Only for debugging/investigating.
83const OPTION_TRACE: u32 = 1 << 0;
84/// When iterating over all matches within a text (e.g. with `find_iter`), empty matches need to be
85/// handled specially. If we kept matching at the same position, we'd never stop. So what we do
86/// after we've had an empty match, is to advance the position where matching is attempted.
87/// If `\G` is used in the pattern, that means it no longer matches. If we didn't tell the VM about
88/// the fact that we skipped because of an empty match, it would still treat `\G` as matching. So
89/// this option is for communicating that to the VM. Phew.
90pub(crate) const OPTION_SKIPPED_EMPTY_MATCH: u32 = 1 << 1;

95/// Instruction of the VM.
96#[derive(Debug, Clone)]
97pub enum Insn {
98    /// Successful end of program
99    End,
100    /// Match any character (including newline)
101    Any,
102    /// Match any character (not including newline)
103    AnyNoNL,
104    /// Match the literal string at the current index
105    Lit(String), // should be cow?
106    /// Split execution into two threads. The two fields are positions of instructions. Execution
107    /// first tries the first thread. If that fails, the second position is tried.
108    Split(usize, usize),
109    /// Jump to instruction at position
110    Jmp(usize),
111    /// Save the current string index into the specified slot
112    Save(usize),
113    /// Save `0` into the specified slot
114    Save0(usize),
115    /// Set the string index to the value that was saved in the specified slot
116    Restore(usize),
117    /// Repeat greedily (match as much as possible)
118    RepeatGr {
119        /// Minimum number of matches
120        lo: usize,
121        /// Maximum number of matches
122        hi: usize,
123        /// The instruction after the repeat
124        next: usize,
125        /// The slot for keeping track of the number of repetitions
126        repeat: usize,
127    },
128    /// Repeat non-greedily (prefer matching as little as possible)
129    RepeatNg {
130        /// Minimum number of matches
131        lo: usize,
132        /// Maximum number of matches
133        hi: usize,
134        /// The instruction after the repeat
135        next: usize,
136        /// The slot for keeping track of the number of repetitions
137        repeat: usize,
138    },
139    /// Repeat greedily and prevent infinite loops from empty matches
140    RepeatEpsilonGr {
141        /// Minimum number of matches
142        lo: usize,
143        /// The instruction after the repeat
144        next: usize,
145        /// The slot for keeping track of the number of repetitions
146        repeat: usize,
147        /// The slot for saving the previous IX to check if we had an empty match
148        check: usize,
149    },
150    /// Repeat non-greedily and prevent infinite loops from empty matches
151    RepeatEpsilonNg {
152        /// Minimum number of matches
153        lo: usize,
154        /// The instruction after the repeat
155        next: usize,
156        /// The slot for keeping track of the number of repetitions
157        repeat: usize,
158        /// The slot for saving the previous IX to check if we had an empty match
159        check: usize,
160    },
161    /// Negative look-around failed
162    FailNegativeLookAround,
163    /// Set IX back by the specified number of characters
164    GoBack(usize),
165    /// Back reference to a group number to check
166    Backref(usize),
167    /// Begin of atomic group
168    BeginAtomic,
169    /// End of atomic group
170    EndAtomic,
171    /// Delegate matching to the regex crate for a fixed size
172    DelegateSized(Box<Regex>, usize),
173    /// Delegate matching to the regex crate
174    Delegate {
175        /// The regex
176        inner: Box<Regex>,
177        /// The same regex but matching an additional character on the left.
178        ///
179        /// E.g. if `inner` is `^\b`, `inner1` is `^(?s:.)\b`. Why do we need this? Because `\b`
180        /// needs to know the previous character to work correctly. Let's say we're currently at the
181        /// second character of the string `xy`. Should `\b` match there? No. But if we'd run `^\b`
182        /// against `y`, it would match (incorrect). To do the right thing, we run `^(?s:.)\b`
183        /// against `xy`, which does not match.
184        ///
185        /// We only need this for regexes that "look left", i.e. need to know what the previous
186        /// character was.
187        inner1: Option<Box<Regex>>,
188        /// The first group number that this regex captures (if it contains groups)
189        start_group: usize,
190        /// The last group number
191        end_group: usize,
192    },
193    /// Anchor to match at the position where the previous match ended
194    ContinueFromPreviousMatchEnd,
195    /// Continue only if the specified capture group has already been populated as part of the match
196    BackrefExistsCondition(usize),
197}

199/// Sequence of instructions for the VM to execute.
200#[derive(Debug, Clone)]
201pub struct Prog {
202    /// Instructions of the program
203    pub body: Vec<Insn>,
204    n_saves: usize,
205}
206
207impl Prog {
208    pub(crate) fn new(body: Vec<Insn>, n_saves: usize) -> Prog {
209        Prog { body, n_saves }
210    }
211
212    #[doc(hidden)]
213    pub(crate) fn debug_print(&self) {
214        for (i, insn) in self.body.iter().enumerate() {
215            println!("{:3}: {:?}", i, insn);
216        }
217    }

220#[derive(Debug)]
221struct Branch {
222    pc: usize,
223    ix: usize,
224    nsave: usize,
225}
226
227#[derive(Debug)]
228struct Save {
229    slot: usize,
230    value: usize,
231}
232
233struct State {
234    /// Saved values indexed by slot. Mostly indices to s, but can be repeat values etc.
235    /// Always contains the saves of the current state.
236    saves: Vec<usize>,
237    /// Stack of backtrack branches.
238    stack: Vec<Branch>,
239    /// Old saves (slot, value)
240    oldsave: Vec<Save>,
241    /// Number of saves at the end of `oldsave` that need to be restored to `saves` on pop
242    nsave: usize,
243    explicit_sp: usize,
244    /// Maximum size of the stack. If the size would be exceeded during execution, a `StackOverflow`
245    /// error is raised.
246    max_stack: usize,
247    options: u32,
248}
249
250// Each element in the stack conceptually represents the entire state
251// of the machine: the pc (index into prog), the index into the
252// string, and the entire vector of saves. However, copying the save
253// vector on every push/pop would be inefficient, so instead we use a
254// copy-on-write approach for each slot within the save vector. The
255// top `nsave` elements in `oldsave` represent the delta from the
256// current machine state to the top of stack.
257
258impl State {
259    fn new(n_saves: usize, max_stack: usize, options: u32) -> State {
260        State {
261            saves: vec![usize::MAX; n_saves],
262            stack: Vec::new(),
263            oldsave: Vec::new(),
264            nsave: 0,
265            explicit_sp: n_saves,
266            max_stack,
267            options,
268        }
269    }
270
271    // push a backtrack branch
272    fn push(&mut self, pc: usize, ix: usize) -> Result<()> {
273        if self.stack.len() < self.max_stack {
274            let nsave = self.nsave;
275            self.stack.push(Branch { pc, ix, nsave });
276            self.nsave = 0;
277            self.trace_stack("push");
278            Ok(())
279        } else {
280            Err(Error::RuntimeError(RuntimeError::StackOverflow))
281        }
282    }
283
284    // pop a backtrack branch
285    fn pop(&mut self) -> (usize, usize) {
286        for _ in 0..self.nsave {
287            let Save { slot, value } = self.oldsave.pop().unwrap();
288            self.saves[slot] = value;
289        }
290        let Branch { pc, ix, nsave } = self.stack.pop().unwrap();
291        self.nsave = nsave;
292        self.trace_stack("pop");
293        (pc, ix)
294    }
295
296    fn save(&mut self, slot: usize, val: usize) {
297        for i in 0..self.nsave {
298            // could avoid this iteration with some overhead; worth it?
299            if self.oldsave[self.oldsave.len() - i - 1].slot == slot {
300                // already saved, just update
301                self.saves[slot] = val;
302                return;
303            }
304        }
305        self.oldsave.push(Save {
306            slot,
307            value: self.saves[slot],
308        });
309        self.nsave += 1;
310        self.saves[slot] = val;
311
312        if self.options & OPTION_TRACE != 0 {
313            println!("saves: {:?}", self.saves);
314        }
315    }
316
317    fn get(&self, slot: usize) -> usize {
318        self.saves[slot]
319    }
320
321    // push a value onto the explicit stack; note: the entire contents of
322    // the explicit stack is saved and restored on backtrack.
323    fn stack_push(&mut self, val: usize) {
324        if self.saves.len() == self.explicit_sp {
325            self.saves.push(self.explicit_sp + 1);
326        }
327        let explicit_sp = self.explicit_sp;
328        let sp = self.get(explicit_sp);
329        if self.saves.len() == sp {
330            self.saves.push(val);
331        } else {
332            self.save(sp, val);
333        }
334        self.save(explicit_sp, sp + 1);
335    }
336
337    // pop a value from the explicit stack
338    fn stack_pop(&mut self) -> usize {
339        let explicit_sp = self.explicit_sp;
340        let sp = self.get(explicit_sp) - 1;
341        let result = self.get(sp);
342        self.save(explicit_sp, sp);
343        result
344    }
345
346    /// Get the current number of backtrack branches
347    fn backtrack_count(&self) -> usize {
348        self.stack.len()
349    }
350
351    /// Discard backtrack branches that were pushed since the call to `backtrack_count`.
352    ///
353    /// What we want:
354    /// * Keep the current `saves` as they are
355    /// * Only keep `count` backtrack branches on `stack`, discard the rest
356    /// * Keep the first `oldsave` for each slot, discard the rest (multiple pushes might have
357    ///   happened with saves to the same slot)
358    fn backtrack_cut(&mut self, count: usize) {
359        if self.stack.len() == count {
360            // no backtrack branches to discard, all good
361            return;
362        }
363        // start and end indexes of old saves for the branch we're cutting to
364        let (oldsave_start, oldsave_end) = {
365            let mut end = self.oldsave.len() - self.nsave;
366            for &Branch { nsave, .. } in &self.stack[count + 1..] {
367                end -= nsave;
368            }
369            let start = end - self.stack[count].nsave;
370            (start, end)
371        };
372        let mut saved = BTreeSet::new();
373        // keep all the old saves of our branch (they're all for different slots)
374        for &Save { slot, .. } in &self.oldsave[oldsave_start..oldsave_end] {
375            saved.insert(slot);
376        }
377        let mut oldsave_ix = oldsave_end;
378        // for other old saves, keep them only if they're for a slot that we haven't saved yet
379        for ix in oldsave_end..self.oldsave.len() {
380            let Save { slot, .. } = self.oldsave[ix];
381            let new_slot = saved.insert(slot);
382            if new_slot {
383                // put the save we want to keep (ix) after the ones we already have (oldsave_ix)
384                // note that it's fine if the indexes are the same (then swapping is a no-op)
385                self.oldsave.swap(oldsave_ix, ix);
386                oldsave_ix += 1;
387            }
388        }
389        self.stack.truncate(count);
390        self.oldsave.truncate(oldsave_ix);
391        self.nsave = oldsave_ix - oldsave_start;
392    }
393
394    #[inline]
395    fn trace_stack(&self, operation: &str) {
396        if self.options & OPTION_TRACE != 0 {
397            println!("stack after {}: {:?}", operation, self.stack);
398        }
399    }
400}

402fn codepoint_len_at(s: &str, ix: usize) -> usize {
403    codepoint_len(s.as_bytes()[ix])
404}
405
406#[inline]
407fn matches_literal(s: &str, ix: usize, end: usize, literal: &str) -> bool {
408    // Compare as bytes because the literal might be a single byte char whereas ix
409    // points to a multibyte char. Comparing with str would result in an error like
410    // "byte index N is not a char boundary".
411    end <= s.len() && &s.as_bytes()[ix..end] == literal.as_bytes()
412}

424/// Run the program with options.
425#[allow(clippy::cognitive_complexity)]
426pub(crate) fn run(
427    prog: &Prog,
428    s: &str,
429    pos: usize,
430    option_flags: u32,
431    options: &RegexOptions,
432) -> Result<Option<Vec<usize>>> {
433    let mut state = State::new(prog.n_saves, MAX_STACK, option_flags);
434    if option_flags & OPTION_TRACE != 0 {
435        println!("pos\tinstruction");
436    }
437    let mut backtrack_count = 0;
438    let mut pc = 0;
439    let mut ix = pos;
440    loop {
441        // break from this loop to fail, causes stack to pop
442        'fail: loop {
443            if option_flags & OPTION_TRACE != 0 {
444                println!("{}\t{} {:?}", ix, pc, prog.body[pc]);
445            }
446            match prog.body[pc] {
447                Insn::End => {
448                    // save of end position into slot 1 is now done
449                    // with an explicit group; we might want to
450                    // optimize that.
451                    //state.saves[1] = ix;
452                    if option_flags & OPTION_TRACE != 0 {
453                        println!("saves: {:?}", state.saves);
454                    }
455                    if let Some(&slot1) = state.saves.get(1) {
456                        // With some features like keep out (\K), the match start can be after
457                        // the match end. Cap the start to <= end.
458                        if state.get(0) > slot1 {
459                            state.save(0, slot1);
460                        }
461                    }
462                    return Ok(Some(state.saves));
463                }
464                Insn::Any => {
465                    if ix < s.len() {
466                        ix += codepoint_len_at(s, ix);
467                    } else {
468                        break 'fail;
469                    }
470                }
471                Insn::AnyNoNL => {
472                    if ix < s.len() && s.as_bytes()[ix] != b'\n' {
473                        ix += codepoint_len_at(s, ix);
474                    } else {
475                        break 'fail;
476                    }
477                }
478                Insn::Lit(ref val) => {
479                    let ix_end = ix + val.len();
480                    if !matches_literal(s, ix, ix_end, val) {
481                        break 'fail;
482                    }
483                    ix = ix_end;
484                }
485                Insn::Split(x, y) => {
486                    state.push(y, ix)?;
487                    pc = x;
488                    continue;
489                }
490                Insn::Jmp(target) => {
491                    pc = target;
492                    continue;
493                }
494                Insn::Save(slot) => state.save(slot, ix),
495                Insn::Save0(slot) => state.save(slot, 0),
496                Insn::Restore(slot) => ix = state.get(slot),
497                Insn::RepeatGr {
498                    lo,
499                    hi,
500                    next,
501                    repeat,
502                } => {
503                    let repcount = state.get(repeat);
504                    if repcount == hi {
505                        pc = next;
506                        continue;
507                    }
508                    state.save(repeat, repcount + 1);
509                    if repcount >= lo {
510                        state.push(next, ix)?;
511                    }
512                }
513                Insn::RepeatNg {
514                    lo,
515                    hi,
516                    next,
517                    repeat,
518                } => {
519                    let repcount = state.get(repeat);
520                    if repcount == hi {
521                        pc = next;
522                        continue;
523                    }
524                    state.save(repeat, repcount + 1);
525                    if repcount >= lo {
526                        state.push(pc + 1, ix)?;
527                        pc = next;
528                        continue;
529                    }
530                }
531                Insn::RepeatEpsilonGr {
532                    lo,
533                    next,
534                    repeat,
535                    check,
536                } => {
537                    let repcount = state.get(repeat);
538                    if repcount > lo && state.get(check) == ix {
539                        // prevent zero-length match on repeat
540                        break 'fail;
541                    }
542                    state.save(repeat, repcount + 1);
543                    if repcount >= lo {
544                        state.save(check, ix);
545                        state.push(next, ix)?;
546                    }
547                }
548                Insn::RepeatEpsilonNg {
549                    lo,
550                    next,
551                    repeat,
552                    check,
553                } => {
554                    let repcount = state.get(repeat);
555                    if repcount > lo && state.get(check) == ix {
556                        // prevent zero-length match on repeat
557                        break 'fail;
558                    }
559                    state.save(repeat, repcount + 1);
560                    if repcount >= lo {
561                        state.save(check, ix);
562                        state.push(pc + 1, ix)?;
563                        pc = next;
564                        continue;
565                    }
566                }
567                Insn::GoBack(count) => {
568                    for _ in 0..count {
569                        if ix == 0 {
570                            break 'fail;
571                        }
572                        ix = prev_codepoint_ix(s, ix);
573                    }
574                }
575                Insn::FailNegativeLookAround => {
576                    // Reaching this instruction means that the body of the
577                    // look-around matched. Because it's a *negative* look-around,
578                    // that means the look-around itself should fail (not match).
579                    // But before, we need to discard all the states that have
580                    // been pushed with the look-around, because we don't want to
581                    // explore them.
582                    loop {
583                        let (popped_pc, _) = state.pop();
584                        if popped_pc == pc + 1 {
585                            // We've reached the state that would jump us to
586                            // after the look-around (in case the look-around
587                            // succeeded). That means we popped enough states.
588                            break;
589                        }
590                    }
591                    break 'fail;
592                }
593                Insn::Backref(slot) => {
594                    let lo = state.get(slot);
595                    if lo == usize::MAX {
596                        // Referenced group hasn't matched, so the backref doesn't match either
597                        break 'fail;
598                    }
599                    let hi = state.get(slot + 1);
600                    if hi == usize::MAX {
601                        // Referenced group hasn't matched, so the backref doesn't match either
602                        break 'fail;
603                    }
604                    let ref_text = &s[lo..hi];
605                    let ix_end = ix + ref_text.len();
606                    if !matches_literal(s, ix, ix_end, ref_text) {
607                        break 'fail;
608                    }
609                    ix = ix_end;
610                }
611                Insn::BackrefExistsCondition(group) => {
612                    let lo = state.get(group * 2);
613                    if lo == usize::MAX {
614                        // Referenced group hasn't matched, so the backref doesn't match either
615                        break 'fail;
616                    }
617                }
618                Insn::BeginAtomic => {
619                    let count = state.backtrack_count();
620                    state.stack_push(count);
621                }
622                Insn::EndAtomic => {
623                    let count = state.stack_pop();
624                    state.backtrack_cut(count);
625                }
626                Insn::DelegateSized(ref inner, size) => {
627                    if inner.is_match(&s[ix..]) {
628                        // We could analyze for ascii-only, and ix += size in
629                        // that case. Unlikely to be speed-limiting though.
630                        for _ in 0..size {
631                            ix += codepoint_len_at(s, ix);
632                        }
633                    } else {
634                        break 'fail;
635                    }
636                }
637                Insn::Delegate {
638                    ref inner,
639                    ref inner1,
640                    start_group,
641                    end_group,
642                } => {
643                    // Note: Why can't we use `find_at` or `captures_read_at` here instead of the
644                    // `inner1` regex? We only want to match at the current location, so our regexes
645                    // need to have an anchor: `^foo` (without `^`, it would match `foo` anywhere).
646                    // But regex like `^foo` won't match in `bar foo` with `find_at(s, 4)` because
647                    // `^` only matches at the beginning of the text.
648                    let re = match *inner1 {
649                        Some(ref inner1) if ix > 0 => {
650                            ix = prev_codepoint_ix(s, ix);
651                            inner1
652                        }
653                        _ => inner,
654                    };
655                    if start_group == end_group {
656                        // No groups, so we can use `find` which is faster than `captures_read`
657                        match re.find(&s[ix..]) {
658                            Some(m) => ix += m.end(),
659                            _ => break 'fail,
660                        }
661                    } else {
662                        let mut locations = re.capture_locations();
663                        if let Some(m) = re.captures_read(&mut locations, &s[ix..]) {
664                            for i in 0..(end_group - start_group) {
665                                let slot = (start_group + i) * 2;
666                                if let Some((start, end)) = locations.get(i + 1) {
667                                    state.save(slot, ix + start);
668                                    state.save(slot + 1, ix + end);
669                                } else {
670                                    state.save(slot, usize::MAX);
671                                    state.save(slot + 1, usize::MAX);
672                                }
673                            }
674                            ix += m.end();
675                        } else {
676                            break 'fail;
677                        }
678                    }
679                }
680                Insn::ContinueFromPreviousMatchEnd => {
681                    if ix > pos || option_flags & OPTION_SKIPPED_EMPTY_MATCH != 0 {
682                        break 'fail;
683                    }
684                }
685            }
686            pc += 1;
687        }
688        if option_flags & OPTION_TRACE != 0 {
689            println!("fail");
690        }
691        // "break 'fail" goes here
692        if state.stack.is_empty() {
693            return Ok(None);
694        }
695
696        backtrack_count += 1;
697        if backtrack_count > options.backtrack_limit {
698            return Err(Error::RuntimeError(RuntimeError::BacktrackLimitExceeded));
699        }
700
701        let (newpc, newix) = state.pop();
702        pc = newpc;
703        ix = newix;
704    }
705}

817    #[derive(Clone, Debug)]
818    enum Operation {
819        Push,
820        Pop,
821        Save(usize, usize),
822    }
823
824    impl Arbitrary for Operation {
825        fn arbitrary(g: &mut Gen) -> Self {
826            match g.choose(&[0, 1, 2]) {
827                Some(0) => Operation::Push,
828                Some(1) => Operation::Pop,
829                _ => Operation::Save(
830                    *g.choose(&[0usize, 1, 2, 3, 4]).unwrap(),
831                    usize::arbitrary(g),
832                ),
833            }
834        }
835    }
836
837    fn check_saves_for_operations(operations: Vec<Operation>) -> bool {
838        let slots = operations
839            .iter()
840            .map(|o| match o {
841                &Operation::Save(slot, _) => slot + 1,
842                _ => 0,
843            })
844            .max()
845            .unwrap_or(0);
846        if slots == 0 {
847            // No point checking if there's no save instructions
848            return true;
849        }
850
851        // Stack with the complete VM state (including saves)
852        let mut stack = Vec::new();
853        let mut saves = vec![usize::MAX; slots];
854
855        let mut state = State::new(slots, MAX_STACK, 0);
856
857        let mut expected = Vec::new();
858        let mut actual = Vec::new();
859
860        for operation in operations {
861            match operation {
862                Operation::Push => {
863                    // We're not checking pc and ix later, so don't bother
864                    // putting in random values.
865                    stack.push((0, 0, saves.clone()));
866                    state.push(0, 0).unwrap();
867                }
868                Operation::Pop => {
869                    // Note that because we generate the operations randomly
870                    // there might be more pops than pushes. So ignore a pop
871                    // if the stack was empty.
872                    if let Some((_, _, previous_saves)) = stack.pop() {
873                        saves = previous_saves;
874                        state.pop();
875                    }
876                }
877                Operation::Save(slot, value) => {
878                    saves[slot] = value;
879                    state.save(slot, value);
880                }
881            }
882
883            // Remember state of saves for checking later
884            expected.push(saves.clone());
885            let mut actual_saves = vec![usize::MAX; slots];
886            for i in 0..slots {
887                actual_saves[i] = state.get(i);
888            }
889            actual.push(actual_saves);
890        }
891
892        expected == actual
893    }
894
895    quickcheck! {
896        fn state_save_quickcheck(operations: Vec<Operation>) -> bool {
897            check_saves_for_operations(operations)
898        }
899    }
