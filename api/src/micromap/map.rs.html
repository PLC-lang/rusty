32/// A faster alternative of [`std::collections::HashMap`].
33///
34/// For example, this is how you make a map, which is allocated on stack and is capable of storing
35/// up to eight key-values pairs:
36///
37/// ```
38/// let mut m : micromap::Map<u64, &str, 8> = micromap::Map::new();
39/// m.insert(1, "Jeff Lebowski");
40/// m.insert(2, "Walter Sobchak");
41/// assert_eq!(m.len(), 2);
42/// assert_eq!(m.capacity(), 8);
43/// ```
44///
45/// It is faster because it doesn't use a hash function at all. It simply keeps
46/// all pairs in an array and when it's necessary to find a value, it goes through
47/// all pairs comparing the needle with each pair available. Also it is faster
48/// because it doesn't use heap. When a [`Map`] is being created, it allocates the necessary
49/// space on stack. That's why the maximum size of the map must be provided in
50/// compile time.
51///
52/// It is also faster because it doesn't grow in size. When a [`Map`] is created,
53/// its size is fixed on stack. If an attempt is made to insert too many keys
54/// into it, it simply panics. Moreover, in the "release" mode it doesn't panic,
55/// but its behaviour is undefined. In the "release" mode all boundary checks
56/// are disabled, for the sake of higher performance.
57pub struct Map<K, V, const N: usize> {
58    /// The next available pair in the array.
59    len: usize,
60    /// The fixed-size array of key-value pairs.
61    pairs: [MaybeUninit<(K, V)>; N],
62}
